# ディレクトリ構造

```text
hikkoshi_lens/
├── .git/ # Git バージョン管理システム用ディレクトリ
├── .gitignore # Git で無視するファイルやディレクトリを指定するファイル
├── backend/ # バックエンドアプリケーション (FastAPI)
│ ├── app/ # FastAPI アプリケーションのコアロジック
│ │ ├── api/ # API エンドポイント定義 (ルーター)
│ │ │ └── v1/ # API バージョン 1
│ │ │ ├── endpoints/ # 各リソースのエンドポイントモジュール
│ │ │ └── schemas/ # Pydantic モデル (データバリデーション、シリアライゼーション)
│ │ │ ├── request/ # リクエストモデル
│ │ │ └── response/ # レスポンスモデル
│ │ ├── core/ # 設定、共通ロジック、セキュリティ関連
│ │ ├── const/ # 定数ファイル (例: prefecture_codes.json)
│ │ ├── crud/ # データベース操作 (Create, Read, Update, Delete)
│ │ ├── db/ # データベース接続、セッション管理 (MongoDB)
│ │ ├── models/ # データベースモデル (Pydantic と連携することが多い)
│ │ ├── services/ # ビジネスロジック、最適化処理 (pulp)
│ │ └── utils/ # ユーティリティ関数
│ ├── tests/ # テストコード
│ │ ├── api/ # APIテスト
│ │ └── services/ # サービステスト
│ ├── .env.example # 環境変数ファイルのテンプレート
│ ├── main.py # FastAPI アプリケーションのエントリーポイント
│ └── requirements.txt # Python ライブラリの依存関係リスト
├── data/ # データ関連ファイル (6. データ要件参照)
│ ├── raw/ # 生データ (ダウンロードしたままのデータ)
│ │ ├── geojson/ # 地理情報
│ │ ├── csv/ # CSV形式のデータ
│ │ └── json/ # JSON形式のデータ
│ └── processed/ # 前処理・クリーニング済みデータ
│   └── hdf5/ # HDF5形式のデータ (Pandas で処理しやすい)
├── docs/ # プロジェクトドキュメント
│ ├── 要件定義書.md # (既存のファイル)
│ └── ディレクトリ構造.md # このファイル
├── docker-compose.yml # Docker Compose 設定ファイル (7. システム構成参照)
├── Dockerfile # バックエンド・フロントエンド用 Dockerfile
├── frontend/ # フロントエンドアプリケーション (Next.js)
│ ├── app/ # Next.js App Router (推奨)
│ │ └── api/ # フロントエンド側の API ルートハンドラ (Next.js API Routes)
│ ├── components/ # 再利用可能な UI コンポーネント (例: Mapbox コンポーネント)
│ │ └── ui/ # UIコンポーネント
│ ├── contexts/ # React Context API 用
│ ├── hooks/ # カスタム React フック
│ ├── lib/ # ライブラリ、ヘルパー関数 (例: API クライアント)
│ ├── public/ # 静的ファイル (画像、フォントなど)
│ ├── styles/ # グローバル CSS、Tailwind CSS 設定 (tailwind.config.js など)
│ ├── types/ # TypeScript 型定義
│ ├── .env.local.example # 環境変数ファイルのテンプレート (ローカル用)
│ ├── next.config.mjs # Next.js 設定ファイル
│ ├── package.json # npm/yarn パッケージ管理ファイル
│ └── tsconfig.json # TypeScript 設定ファイル
└── README.md # プロジェクトの概要、セットアップ手順など
```
